Blockly Reference

Blockly is a visual programming interface that uses colorful blocks to approximate coding concepts in an easy to use manner and that requires no programming experience. Each Block represents a different action or function that you can chain together to create new behaviors in your app. Blocks are in a variety of different shapes and the indents indicate where and how you can connect blocks with each other. Additionally, a yellow highlight appears at potential connection points when you drag one block to hover over another block.

All blocks supported in SOTI Snap are available in the Toolbox on the left of the Blockly Editor. Click a Block category to open it and pull a block from that section. Hover over each block to see a tooltip that briefly describes its purpose.

This set of blocks specifies that textbox194 should be automatically populated with data from the Username and PhoneNumber textboxes, separated with a colon :.

Note: SOTI customized Blockly to blend seamlessly with the SOTI Snap app building process. For a comprehensive overview of Blockly in general, see the Google Blockly Reference Guide.

You can also visit SOTI Blockly Documentation for more examples of Blockly in action.

Tip: Use the search field in the Blockly Editor to quickly find blocks.
Toolbox Category Description
Widget Values Contains generic get and set blocks for various widgets in your SOTI Snap app. Get blocks pull in data from the specified widget while Set blocks dictate what should appear inside a widget, whether that's a fixed value or a dynamic one, created by combining two or more other values or blocks. You can also set the background color of some widgets.

You can set and get values for the following widgets:

  • Textbox
  • Radio Button
  • Checkbox
  • Rating
  • Date
  • Button
  • Layout
  • Section
Note: Widgets do not all support the same set/get values.
Environment Variables Contains any app environment variables defined for this app. You can perform get and set on environment variables.

Read App Environment Variables for more information.

Note: If you haven't created any app environment variables yet, you won't see this toolkit option.
App Actions Contains blocks that perform actions specific to a SOTI Snap app. Currently, only the Navigate block is supported, which redirects users to another location in the SOTI Snap app.
Connections Contains blocks for sending RESTful requests to third party services. All RESTful methods types (GET, POST, PUT, DELETE, HEAD, CONNECT, OPTIONS, TRACE) are supported.

Connection blocks include hooks to determine the effect of successful or failed requests. You can check if your RESTful service is working correctly by click the Test button.

Learn more about RESTful services at MDN REST documentation

Note: The Connections toolbox has been temporarily discontinued. Existing Connections blocks will continue to work but you cannot add new ones to your app. If you edit or delete your existing blocks, they will stop working as well.
Logic Contains blocks that execute based on whether particular conditions are met. The If block only runs its nested statement if the condition specified in the hexagonal connection point is met. Once that condition is met, the logic block will perform whichever action you specified.

You can click the button on an If block to add an Else statement. That is, if the condition is not met, the block will execute an entirely different action.

Note: All blocks that require a condition are indicated by the hexagonal shape.
Loops Contains blocks that run other codeblocks repeatedly until a (user-defined) number of repetitions has occurred or another condition is met. You can even set them to run indefinitely.
Math Contains blocks that can calculate values and perform math operations such as addition, subtraction, multiplication, division and more. Math blocks can also generate random numbers or may contain mathematical constants such as pi π or phi φ .
Text Contains blocks that can manipulate text fragments or 'strings' in your app. You can join multiple text strings together, search for a specific quote within a larger piece of text, change the case of text or display messages.
Lists Contains blocks that can combine multiple values into a single block, which you can then perform operations on. For example, you can create a list of values and then set them to order alphabetically using a Sort block. You can also retrieve a specific item in a list by referencing its position.
Local Variables Allows you to store values, whether of a calculation or another computed result, that you can reuse elsewhere in your Blockly script.
Debugging Allows you to add a Log block to other blocks which can help you troubleshoot your Blockly script. Information gathered from the Log block appears in the browser console while an app is in Preview mode. Right-click and select Inspect / Inspect Element (depending on your browser) and then click the Console tab in the Developer Settings to see the log output.

The log block also sends messages to the SOTI Snap app's error log when you test the app on a device.
